＊ きーわーどを使って３ [修行]

Subversionで＄Ｉｄキーワードを使うには
  $ cd work/trunk
  $ svn proplist -v ChangeLog
  $ svn propset svn:keywords Id ChangeLog
  property 'svn:keywords' set on 'ChangeLog'
  $ svn proplist -v ChangeLog
  Properties on 'ChangeLog':
  svn:keywords : Id
  $ svn status
  ? trunk
  M ChangeLog

＊ 発注データ、一元化の必要性 [ひとりごと]

設備製作の部品管理こそデータの一元化が必要だ。
発注リスト管理・発注管理・受け入れ管理。
モノが入ったか、入ってないか、本当に発注されているのか、でトラブっていることが多過ぎる。
これは機械の技術開発本来の仕事から外れている。
最低でも受け入れ現場全部にデータ入力端末が必要だ。
機械設計時の部品部番はシンプルにした方が分かりやすいのではないか。
しっかりした組図、シンプルな部品表(部品リスト)、徹底した原始的な発注受け入れ処理。
